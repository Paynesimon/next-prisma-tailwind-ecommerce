# 白标电商系统 — 功能总结与使用指南

## 一、项目概览

基于 Next.js 14 的**白标电商 monorepo**，支持多客户独立建站。

| 应用 | 路径 | 默认端口 | 职责 |
|------|------|----------|------|
| **Storefront** | `apps/storefront` | 7777 | 前台商城、博客、留言 |
| **Admin** | `apps/admin` | 8888 | 后台管理、内容审核 |
| **Provisioning** | `apps/storefront/main.js` | — | 飞书 → Neon + Vercel 开站 |
| **Sync** | `apps/storefront/sync-client.js` | — | 仅同步商品/Banner，不重新部署 |

数据流：

```
飞书多维表格 → main.js / sync-client.js → Neon DB + config.json → Vercel 部署
                                                      ↓
                                            Storefront / Admin 读取
```

## 二、已实现功能总览

### 1. 三套前台主题

在飞书店铺表配置 `主题`：`shop` / `corporate` / `blog`

| 主题 | 风格 | 首页特点 |
|------|------|----------|
| shop | 电商风 | 商品网格、轮播 |
| corporate | 企业风 | 品牌故事、服务导向 |
| blog | 博客风 | 文章列表、精选推荐 |

内页通过 `ThemeMain` 统一适配主题样式。

### 2. 商品详情页

路径：`/products/[productId]`

```
┌─────────────────────────────────────┐
│  图片轮播  │  标题 / 品牌 / 分类      │
│            │  价格 / MOQ / 阶梯价    │
│            │  加购 / 收藏 / 询价      │
├─────────────────────────────────────┤
│  商品简介（商品描述）                │
│  规格参数（规格参数 字段）           │
│  图文详情（详情HTML 字段）           │
├─────────────────────────────────────┤
│  商品评价（可开关）                  │
└─────────────────────────────────────┘
```

### 3. B2B 批发能力（MOQ）

- 飞书字段 `起订量` → `metadata.moq`
- 飞书字段 `批次价格` → `metadata.priceTiers`
- 加购、购物车、结算均校验 MOQ
- 阶梯价表格展示在详情页

### 4. 互动反馈系统

| 功能 | 前台 | 后台审核 | 飞书开关 |
|------|------|----------|----------|
| 商品评价 | 详情页 + 卡片评分 | `/reviews` | `开启商品评价` |
| 博客评论 | 文章底部 | `/blog-comments` | `开启博客评论` |
| 联系留言 | `/contact` + 商品询价 | `/messages` | `开启联系留言` |

规则：

- **商品评价**：须已购买（B2B 验单）
- **博客评论**：游客可留，默认待审核
- **联系留言**：无需登录，新留言邮件通知店主

### 5. 博客

- MDX 文章，`/blog` 列表 + `/blog/[slug]` 详情
- blog 主题有独立列表布局
- 评论可开关

### 6. 飞书自动化

| 脚本 | 何时用 | 效果 |
|------|--------|------|
| `node main.js <客户ID>` | 新客户开站 / 改店名主题 | 建库、推 schema、写 config、部署 Vercel |
| `node sync-client.js <客户ID>` | 日常改商品 | 约 1 分钟同步 DB，**无需重新部署** |
| `node db-push-client.js <客户ID>` | Schema 变更后 | 对客户 Neon 库执行 `prisma db push` |
| `node cron.js` | 定时任务 | 自动 sync |

### 7. 前台语言包（i18n）

- **一店一语言**：飞书 `默认语言` 写入 `config.json`，整站 UI 使用该语言
- **语言包路径**：`apps/storefront/src/i18n/messages/*.json`
- **Admin 后台**：界面固定中文；客户外语留言/评价/评论 **原文展示**，供中文运营审核

**已完整翻译的 UI 语言：**

| 代码 | 语言 | 飞书写法示例 |
|------|------|-------------|
| `en` | 英语 | `en` / `English` |
| `zh-CN` | 简体中文 | `中文` / `zh-CN` |
| `ja` | 日语 | `ja` / `日语` |
| `ru` | 俄语 | `ru` / `俄语` |
| `es` | 西班牙语 | `es` / `西班牙语` |
| `pt` | 葡萄牙语 | `pt` / `葡萄牙语` |
| `ar` | 阿拉伯语 | `ar` / `阿拉伯语`（自动 RTL 排版） |

未提供语言包的语言（如 `th`、`vi`）会 **回退英文 UI**；商品/店铺内容仍可用当地语言填写。

**目标地区预设（`目标地区` / `地区` / `市场`）：**

| 代码 | 市场 | 默认语言 | 默认货币 |

## 三、飞书字段配置手册

> 完整客户资料清单见 [客户开站资料清单.html](./客户开站资料清单.html)

### 店铺表

| 字段 | 必填 | 说明 |
|------|------|------|
| 客户ID | ✅ | 唯一标识，与商品表一致 |
| 店铺名称 | ✅ | 改名需 **重新部署** |
| 主题 | 选填 | `shop` / `corporate` / `blog`，默认 shop |
| 开启商品评价 | 选填 | 是/否，默认是 |
| 开启博客评论 | 选填 | 是/否，默认是 |
| 开启联系留言 | 选填 | 是/否，默认是 |
| 目标地区 | 选填 | `US`/`EU`/`CN`/`SEA`/`JP`/`RU`/`LATAM`/`ME`/`AF`/`CA`，见 §7 |
| 默认语言 | 选填 | `en`/`zh-CN`/`ja`/`ru`/`es`/`pt`/`ar` 等，决定前台 UI 语言 |
| 货币 / 时区 | 选填 | 覆盖地区预设，如 `VND`、`Asia/Bangkok` |

### 商品表

| 字段 | 必填 | 说明 |
|------|------|------|
| 客户ID | ✅ | 须与店铺表一致 |
| 商品名称 | ✅ | 改名 = 新商品 |
| 价格 | ✅ | 数字 |
| 图片URL | ✅ | HTTPS 公网链接 |
| 品牌 | 选填 | 详情页展示，可筛选 |
| 分类 | 选填 | 逗号分隔，可筛选 |
| 商品描述 | 选填 | 详情页「商品简介」 |
| 关键词 | 选填 | SEO |
| 是否上架 | 选填 | 默认上架 |
| 起订量 | 选填 | 如 `10`，MOQ 校验 |
| 批次价格 | 选填 | `100:9.5,500:8.0`（数量:单价） |
| 规格参数 | 选填 | `材质:不锈钢,尺寸:10×20cm` |
| 详情HTML | 选填 | 富文本 HTML；也可用字段名 `商品详情HTML` |

**规格参数格式：**

```
材质:304不锈钢
尺寸:10×20cm
重量:500g
```

或一行：`材质:304不锈钢,尺寸:10×20cm,重量:500g`

**详情HTML示例：**

```html
<h3>产品特点</h3>
<p>适用于工业级应用场景……</p>
<img src="https://example.com/detail.jpg" alt="细节图" />
```

### Banner 表（选填）

| 字段 | 说明 |
|------|------|
| 客户ID | 必填 |
| 图片URL | HTTPS 公网图链 |
| 绑定分类 | 留空=首页轮播；填写=该分类封面 |

## 四、本地开发指南

### 1. 环境准备

```bash
cd apps/storefront && npm install
cd apps/admin && npm install
```

### 2. 配置 `.env`

`apps/storefront/.env` 关键项：

```env
DATABASE_URL="postgresql://...@xxx-pooler....neon.tech/neondb?sslmode=require&connect_timeout=30"
JWT_SECRET_KEY="..."
NEXT_PUBLIC_SITE_URL="http://localhost:7777"
```

建议：

- 使用 Neon **Pooler** 端点（主机名含 `-pooler`）
- 加 `connect_timeout=30`，缓解冷启动超时

`apps/admin/.env` 需单独配置 `DATABASE_URL`。本地测试审核功能时，**建议与 storefront 指向同一库**。

### 3. 同步数据库 Schema

```bash
cd apps/storefront && npx prisma db push
cd apps/admin && npx prisma db push
```

### 4. 启动服务

```bash
# 终端 1 — 前台
cd apps/storefront && npm run dev    # http://localhost:7777

# 终端 2 — 后台
cd apps/admin && npm run dev         # http://localhost:8888
```

### 5. 常用页面自测

| 页面 | URL |
|------|-----|
| 首页 | `/` |
| 商品列表 | `/products` |
| 商品详情 | `/products/{id}` |
| 购物车 | `/cart` |
| 联系留言 | `/contact` |
| 博客列表 | `/blog` |
| 博客详情 | `/blog/{slug}` |
| 后台留言 | `http://localhost:8888/messages` |

## 五、运营工作流

### 新客户开站

1. 飞书填好：店铺表 + 商品表（+ Banner、支付，可选）
2. `node main.js client001`
3. 等待：建 Neon 库 → db push → 写 config → 部署 Vercel
4. 访问分配的域名验收

### 日常维护

| 改什么 | 怎么做 | 是否需要重新部署 |
|--------|--------|------------------|
| 商品名称/价格/图片/规格/详情/MOQ | `node sync-client.js client001` | ❌ 不需要 |
| 店铺名称 / 主题 / 功能开关 | 改飞书后 `main.js` 或改 config + 部署 | ✅ 需要 |
| Prisma schema 变更 | `prisma db push`（每个客户库） | 视情况 |
| 支付配置 | 飞书支付表 + 重部署 | ✅ 需要 |

### 后台审核流程

1. 用户提交评价 / 评论 / 留言 → 状态为 **Pending** 或 **New**
2. 登录 Admin → 对应管理页
3. 操作：通过 / 拒绝 / 已读 / 已回复 / 归档
4. 通过后前台才展示（评价、评论）

## 七、常见问题

### 1. `Can't reach database server`（Neon）

**原因：** Neon 免费版休眠，冷启动需 5–20 秒。

**处理：**

```bash
cd apps/storefront && npx prisma db push
```

等命令完成后再刷新页面。使用 Pooler + `connect_timeout=30` 可降低概率。

### 2. `Failed to fetch`（评价区）

多为 Strict Mode 请求被取消或 DB 未就绪。硬刷新（Cmd+Shift+R）即可。

### 3. Admin 看不到前台测试数据

两个 app 的 `DATABASE_URL` 指向了不同 Neon 库。本地测试请统一为同一库。

### 4. 改了飞书商品但网站没变

确认已运行 `sync-client.js`，且 `客户ID` 与店铺表一致。改店名/主题需重新部署。

### 5. 商品详情区块不显示

检查数据库 `metadata` 是否有数据，或飞书是否填写了 `规格参数` / `详情HTML`。仅有 `商品描述` 时只显示「商品简介」。

## 九、部署说明

- Storefront 与 Admin 在 Vercel 上作为**两个独立项目**部署
- Root Directory 分别为 `apps/storefront` 和 `apps/admin`
- Schema 变更需对每个客户库执行 `npx prisma db push` 或 `node db-push-client.js <客户ID>`
- 飞书功能开关、主题变更需重新部署；商品/MOQ/规格/详情通过 sync 即可更新
