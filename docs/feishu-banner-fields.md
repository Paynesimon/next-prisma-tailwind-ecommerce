# 飞书 Banner 表字段说明

用于 **首页 Hero 轮播** 与 **分类卡片封面** 配图。表 ID：`tblBXnPE0tUz4xut`（与 `main.js` / `sync-client.js` 中 `FEISHU_TABLES.banners` 一致）。

## 客户隔离（重要）

Banner 与商品一样，**每一行都必须填 `客户ID`**，且与店铺表、商品表保持一致（如 `client001`）。

同步时的隔离机制：

1. **飞书侧**：`main.js` / `sync-client.js` 只读取 `客户ID === 当前建站客户` 的 Banner 行（`bannerRecordsForClient`）。
2. **数据库侧**：每个客户有独立 Neon 库（`shop-{客户ID}`），即使飞书填错，也不会写到其他客户库里。
3. **前台侧**：只连接该客户自己的数据库，不会看到其他客户的 Banner。

**注意**：`客户ID` 留空或填错的行，不会进入任何客户站点；填成别人的 ID 则只会进那个客户的站。

## 字段一览

| 字段名 | 必填 | 类型 | 说明 |
|--------|------|------|------|
| **客户ID** | 是 | 文本 | **每行必填**，与店铺/商品表一致，如 `client001`；用于多客户共用一张 Banner 表时的数据隔离 |
| **图片URL** | 是 | URL | HTTPS 公网图片地址 |
| **标签名** | 否 | 文本 | 内部备注或无障碍描述，前台一般不展示 |
| **排序** | 否 | 数字 | 越小越靠前；首页轮播与分类封面各自按此字段排序后写入 |
| **绑定分类** | 否 | 文本 | 见下方规则；**留空 = 首页轮播** |

## `绑定分类` 规则

| 填写方式 | 前台效果 |
|----------|----------|
| **留空** | 仅出现在 **首页 Hero 轮播**，不参与分类卡片 |
| **填写单个分类名** | 仅作为该 **分类卡片封面**（如 `音响`），不出现在首页轮播 |
| **多个分类** | 用英文或中文逗号分隔，如 `音响,耳机`；同一张图会绑定到多个分类封面 |

**分类名必须与商品表「分类」字段完全一致**（区分大小写与空格）。同步时会自动 `upsert` 对应分类记录。

若某分类没有绑定 Banner，首页分类区会 **回退到该分类下商品首图**（优先精选、再按上架时间）。

## 同步行为

- **一键建站**（`main.js`）：全量写入 Neon，`Banner` ↔ `Category` 多对多关联。
- **日常同步**（`sync-client.js`）：同样逻辑，**无需重新部署**；刷新前台即可看到分类封面变化。
- **config.json**：仅写入 **未绑定分类** 的 Banner，供无数据库时的构建回退。

同步日志示例：

```text
✅ Banner 同步完成：5 条（首页轮播 3，分类封面 2）
```

## 在飞书表格中新增列

1. 打开 Banner 多维表格 → **添加字段**
2. 字段名：`绑定分类`
3. 类型：**文本**（单行即可）
4. 已有轮播图行保持该列为空；需配分类封面的行填入分类名

无需改代码或重新建站，下次 `sync-client.js` 或 webhook 同步即可生效。

## 常见问题

**Q：同一张图既想轮播又想当分类封面？**  
A：当前设计二选一。请在飞书填两行（同一 `图片URL`），一行留空 `绑定分类`，一行填分类名。

**Q：改了 `绑定分类` 没变化？**  
A：确认 `客户ID` 正确，并执行 `node sync-client.js <客户ID>` 或等待 webhook 同步。

**Q：分类名填了但不显示？**  
A：检查是否与商品「分类」完全一致；该分类下需至少有 1 个商品才会出现在首页分类区。
