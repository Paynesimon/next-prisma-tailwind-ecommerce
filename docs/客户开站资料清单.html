客户开站资料清单

按飞书四张表整理 · 必填 / 选填 / 系统自动生成 / 建站后填写

一、开站前确认

项目要求示例
客户 ID必填，全局唯一，英文小写client001
店铺名称必填TechGear Store
管理员邮箱必填，后台 OTP 登录admin@example.com
联系邮箱建议填展示在联系页
至少 1 个商品名称、价格、图片—
自定义域名选填，建站自动绑 Vercelshop.example.com

客户 ID 确定后勿改，用于数据库、GitHub 仓库、Vercel 项目命名。

二、店铺信息表

客户必填

字段说明
客户ID唯一标识 client003
店铺名称网站标题
店铺简介首页/SEO 描述
管理员邮箱后台登录

建议填写

关键词逗号分隔，SEO 用
主题shop / corporate / blog，默认 shop
目标地区US/EU/UK/CN/SEA/JP/RU/LATAM/ME/AF/CA，决定默认货币与时区
默认语言en/zh-CN/ja/ru/es/pt/ar；ar 自动 RTL
货币 / 时区可覆盖地区预设
联系邮箱 / 创建者客服与品牌名
开启商品评价是/否，默认是；关闭后隐藏评价与卡片评分
开启博客评论是/否，默认是
开启联系留言是/否，默认是；关闭后仅显示邮箱/WhatsApp
自定义域名需客户能改 DNS

系统自动回写（勿手填）

前台/后台地址、部署状态、Sitemap、Google/Bing收录、SEO待办、DNS说明

三、商品表

每件商品一行，客户ID 须与店铺表一致。改商品走 sync，约 1 分钟生效，无需重新部署。

字段说明
客户ID归属客户
商品名称改名=新商品
价格数字
图片URLHTTPS 公网图链
品牌详情页展示，可按品牌筛选
分类逗号分隔，可按分类筛选
商品描述详情页「商品简介」，可 AI 生成
关键词逗号分隔，SEO 用
是否上架默认上架
起订量如 10；购物车/结算校验 MOQ
批次价格数量:单价，逗号或换行分隔，如 100:9.5,500:8.0
规格参数参数名:值，如 材质:不锈钢,尺寸:10×20cm
详情HTML富文本 HTML 详情；字段名也可用「商品详情HTML」

详情页结构：上方购买区 → 简介 / 规格 / 图文详情 → 商品评价（可开关）。详见 docs/使用指南.md

四、Banner 表（选填）

字段说明
客户ID每行必填，须与店铺/商品表一致（多客户共用表时用于隔离）
图片URLHTTPS 公网图链
标签名选填，内部备注
排序数字越小越靠前
绑定分类留空=首页轮播；填写=该分类卡片封面（须与商品「分类」一致）

无 Banner 时用渐变 Hero。分类无专属图时用该分类商品首图。详见 docs/feishu-banner-fields.md

五、支付配置表（选填）

字段说明
客户ID必填
Stripe 公钥/私钥pk_... / sk_...
PayPal Client ID若用 PayPal

不填可浏览，支付不可用。上线后需配 Stripe Webhook。

六、客户素材汇总 & 最小配置

  • 客户 ID（英文，唯一）
  • 店铺名称 + 简介
  • 管理员邮箱（能收验证码）
  • 商品清单（名称、价格、图片）
  • Logo / Banner 图（HTTPS）
  • 自定义域名（可选）
  • Stripe 密钥（若要收款）
  • 社交媒体链接（可选）

最小可建站（快速测试）

表最少字段
店铺表客户ID、店铺名称、店铺简介、管理员邮箱
商品表客户ID、商品名称、价格、图片URL（≥1条）

图片须 https:// 公网链接，勿用本地路径。改商品用 sync；改店名等需 build 重新部署。

七、建站流程

  1. 飞书填好店铺 + 商品（+ 支付，可选）
  2. 触发一键建站（action: build）
  3. 等待约 20–30 分钟
  4. 飞书自动回写前台/后台地址
  5. 有域名 → 按 DNS 说明解析
  6. 点「Google收录」提交 sitemap
  7. 日常改商品 → 自动 sync（约 1 分钟）

八、建站后手动步骤
